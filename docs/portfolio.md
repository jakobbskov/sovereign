# Sovereign application portfolio

## Overview

The Sovereign ecosystem consists of several independent self-hosted applications.

Each application focuses on a specific domain of everyday life. Together they form a small ecosystem of practical tools built around the same architectural philosophy.

The goal is not to build a single monolithic system, but a set of focused applications that remain understandable and independently maintainable.

## Application domains

The current application family consists of three primary domains:

| Domain | Application | Purpose |
|------|------|------|
| Home environment | Sovereign Planta | Plant care and household maintenance |
| Personal finance | Sovereign Finance | Financial overview and decision support |
| Physical training | Sovereign Strength | Strength training and progression tracking |

Each application operates independently but follows the same design principles.

## Portfolio philosophy

The portfolio follows several guiding ideas:

- each application solves a clearly bounded problem
- applications remain operationally independent
- data structures remain transparent
- the system avoids unnecessary coupling
- tools should remain durable over time

This approach allows the ecosystem to grow gradually without creating unnecessary architectural complexity.

## Ecosystem direction

Future applications may extend the Sovereign ecosystem into additional areas of everyday life.

Potential domains include:

- knowledge management
- personal decision logs
- household infrastructure
- long-term planning tools

Expansion should remain consistent with the core principles:

- simplicity
- explainability
- self-hosted operation
- practical utility
