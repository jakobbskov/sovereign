## Principles

- local-first
- self-hosted
- no tracking
- transparent logic
Sovereign is a family of local-first, self-hosted applications designed to support everyday life through calm, understandable, and user-controlled software.

The project is built around a simple idea: important personal tools should remain transparent, portable, inspectable, and independent of unnecessary platform dependence.

## Purpose

The Sovereign project exists to create a small ecosystem of self-hosted tools that are:

- understandable
- durable
- privacy-respecting
- operationally simple
- controlled by the user rather than by platform incentives

The goal is not to build a generic productivity suite, but a set of focused applications with readable logic and long-term practical value.

## Core principles

All Sovereign applications are intended to follow these principles:

- local-first
- self-hosted
- no tracking
- explainable logic
- practical over theatrical
- low operational complexity
- long-term maintainability

## Shared architecture philosophy

The applications are independent repositories, but they share a common architectural philosophy:

- simple repository structure
- explicit documentation
- transparent data models
- operational separation where useful
- self-hosted deployment
- minimal hidden state

## Hosting approach

The Sovereign ecosystem is currently distributed across self-hosted infrastructure.

At a high level:

- Raspberry Pi hosts cloud-facing and proxy-oriented infrastructure
- Beelink hosts selected application services and operational workloads

This split supports both practical self-hosting and controlled service separation.
